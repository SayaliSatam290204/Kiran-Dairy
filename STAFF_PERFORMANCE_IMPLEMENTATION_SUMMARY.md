SALE CREATED\n   └─ POS page → Create sale with shift field (morning/evening)\n   └─ Save to MongoDB Sales collection\n   └─ Include: staffId, shopId, saleDate, shift, items, totalAmount\n\n2. AGGREGATION (On Dashboard Load)\n   └─ Backend receives request for shop performance\n   └─ MongoDB aggregation pipeline groups sales by:\n      ├─ Purchase date (for daily/monthly/yearly)\n      ├─ Shift (morning/evening)\n      └─ Calculates metrics:\n         ├─ totalSales (count)\n         ├─ totalAmount (sum)\n         ├─ itemsSold (array size sum)\n         ├─ avgSaleAmount (totalAmount / totalSales)\n         ├─ staffWorking (unique staffId count per shift)\n         └─ daysWorked (unique dates per shift)\n\n3. API RESPONSE\n   └─ Backend returns structured data:\n      ├─ byShift: [morning {...}, evening {...}]\n      └─ monthlyTotal/yearlyTotal: {...}\n\n4. FRONTEND DISPLAY\n   └─ React receives API response\n   └─ Stores in staffPerformance state\n   └─ Renders shift cards with stat boxes\n   └─ Shows staff count and metrics per shift\n   └─ Displays combined totals\n```\n\n---\n\n## Key Technical Details\n\n### Shift Field Usage\n- **Location:** Sale model in `server/src/models/Sale.js`\n- **Type:** Enum (morning | evening)\n- **Default:** 'morning'\n- **Set in:** POS page when completing sale\n- **Used by:** All aggregation queries for shift breakdown\n\n### Unique Staff Counting\nMongoDB aggregation uses `$addToSet` operator:\n```javascript\n{\n  $group: {\n    _id: '$shift',\n    staffWorking: { $addToSet: '$staffId' }  // Unique staff per shift\n  }\n}\n// Later:\n{\n  $project: {\n    staffWorking: { $size: '$staffWorking' }  // Count them\n  }\n}\n```\n\n### Days Worked Calculation\n```javascript\n{\n  $group: {\n    _id: '$shift',\n    daysWorked: { $addToSet: { \n      $dateToString: { format: '%Y-%m-%d', date: '$saleDate' }\n    }}\n  }\n}\n// Later:\n{\n  $project: {\n    daysWorked: { $size: '$daysWorked' }  // Count unique dates\n  }\n}\n```\n\n### Role-Based Access\n- **Admin:** Can see all staff performance across all shops\n- **Shop Manager:** Can see only their shop's staff performance by shift\n- **Staff:** Cannot access these endpoints\n\n---\n\n## Files Modified
